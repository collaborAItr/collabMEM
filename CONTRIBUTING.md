# Contributing Scope

Thanks for your interest in improving collabMEM Open Architecture.

This repository is documentation-first. Contributions should improve clarity, portability, safety, or practical usefulness for developers building their own cognitive memory layer.

## Good Contributions

- Clarify architecture concepts.
- Add implementation-neutral diagrams.
- Improve privacy or transparency guidance.
- Add examples that are fictional, generic, and easy to adapt.
- Identify unclear wording.
- Suggest safer language around user control or model behavior.
- Improve accessibility of the documentation.

## Out Of Scope

Please do not contribute:

- Production app internals.
- Proprietary prompts.
- Provider-specific routing logic.
- Hidden scoring constants.
- Private database schemas.
- Product-specific endpoints.
- Claims that a software system replicates biological memory.

## Documentation Style

Use plain language. Prefer explanation over hype.

Recommended style:

- Say "inspired by" when discussing cognitive science metaphors.
- Explain tradeoffs.
- Keep examples small.
- Keep implementation choices optional.
- Distinguish required principles from suggested patterns.

Avoid:

- Vendor lock-in.
- Competitive absolutes.
- Unverifiable claims.
- Hidden magic.
- Large code samples that imply this is a library.

## Pull Request Checklist

Before opening a pull request, check:

- The change keeps the project documentation-only.
- The change does not introduce proprietary implementation detail.
- The change works for single-assistant apps, not only multi-assistant systems.
- The change preserves user transparency and manual control.
- The change avoids overclaiming cognitive or biological equivalence.
- Links and markdown headings render correctly.
